# 常用操作案例

> 先看效果，再讲原理。这几个案例覆盖了从游戏、应用到日常工作的典型使用场景。

## 1. 生成一个像素风游戏

直接用自然语言描述想要的游戏，让模型生成完整可运行的代码。

典型 Prompt：

```text
用 HTML + JavaScript 写一个像素风格的贪吃蛇游戏。
要求：16x16 方格，蛇是绿色像素块，食物是红色像素块，
按方向键控制，碰墙或咬到自己结束，显示分数。
不依赖任何外部库，单文件可直接打开运行。
```

演示要点：

- 模型一次生成完整 HTML 文件，复制到浏览器即可运行
- 可以迭代修改：加音效、改速度、换风格
- 体现"把想法直接变成可运行产品"的能力
- 非程序员也能完成，不需要了解代码细节

> 这个案例适合用来建立直觉：AI 不只是问答工具，它可以直接交付可用的东西。

## 2. 工作流

把重复性强、规则清晰的工作交给 AI 自动化处理。

典型场景：

- **会议纪要**：录音转文字，再让模型提炼决策项、待办事项、负责人
- **周报生成**：把工作记录喂给模型，生成结构化周报
- **邮件处理**：批量分类、起草回复、提取关键信息
- **数据整理**：把非结构化文本转成表格或 JSON

工作流的核心逻辑：

```text
原始输入（录音/文档/邮件）
  → 定义清晰的处理规则（Prompt）
  → 模型输出结构化结果
  → 人工确认或直接使用
```

> 工作流不需要写代码，关键是把"我平时怎么做"用清楚的语言告诉模型。

## 3. 生成一个应用

从需求描述到完整 Web 应用，全程用自然语言驱动。

典型 Prompt：

```text
做一个番茄钟应用。
功能：25 分钟工作 + 5 分钟休息循环，开始/暂停/重置按钮，
倒计时数字显示，时间到了有提示音，记录今天完成了几个番茄钟。
风格：简洁，深色背景，单页应用，不依赖外部库。
```

演示要点：

- 完整应用通常 1–3 次迭代即可达到可用状态
- 可以用截图反馈给模型继续调整界面
- 多模态模型可以直接看草图生成代码
- 搭配 Claude Artifacts 或 v0.dev 可以实时预览

## 4. 工作实战演示

把 AI 接入真实工作场景，展示完整操作链路。

演示内容（根据现场实际情况选择）：

| 场景 | 操作链路 |
|------|----------|
| 写分析报告 | 上传原始数据 → 让模型分析趋势 → 生成图表代码 → 输出报告草稿 |
| 处理合同文档 | 上传 PDF → 提取关键条款 → 标注风险点 → 生成摘要 |
| 代码 Review | 粘贴代码 → 让模型找问题 → 解释原因 → 给出修改建议 |
| 制作 PPT 大纲 | 描述主题 → 生成结构 → 逐页展开 → 补充数据和案例 |

现场演示建议：

- 选一个与听众工作最相关的场景
- 展示从"空白"到"有用输出"的完整过程
- 刻意犯一个错，演示如何纠正和迭代
- 让听众提需求，实时演示

> 实战演示的目的不是展示 AI 有多厉害，而是让听众看到"这个东西我明天就能用"。
